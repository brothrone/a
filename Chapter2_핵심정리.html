01 2.1 Requirements
설계요구사항의 성격
  1. 1반복 과정 — 항공기 설계는 iterative process. 첫 번째 설계를 그대로 제작하는 일은 없다. 요구사항도 설계 과정에서 함께 발전한다.
  2. 2firm numbers 필요 — 새 개념을 그리려면 날개 면적, 엔진 크기 등의 구체적 수치가 필요. 이는 특정 요구사항으로부터만 계산 가능
  3. 3고객 중심 요구사항 — 실제 임무 및 성능: 항속거리(Range), 페이로드(Payload), 속도, 저관측성, 주기장 크기 제한 등 포함
  4. 4RFP (군용) — Request for Proposal. 군용기는 고객(군)이 RFP를 통해 최종 요구사항을 "납품물"로 명시. 계약업체와 군 양측의 분석 결과 ★
  5. 5MTOW 결정 — 대부분 항공기의 최대 이륙 총 중량은 페이로드 중량과 항속거리 요구사항으로 결정 → Chap. 3 & 6 "Sizing"
  6. 6엔진 크기·날개 면적 결정 — 최대 속도, 실속 속도, 상승율, 이륙 거리 등 성능 요구사항으로 결정 → Chap. 5
  7. 7비용 목표 — 개발 비용 + 구매 단가 + 운영 비용 포함. 동일 임무를 수행하는 다른 방법보다 비싸면 안 됨
  8. 8규정/규격 — 민항: FAR Part 23 (소형), Part 25 (대형 수송기). 군용: Mil-Specs / Mil-Standards (예: MIL-XX-XXXX)

03 2.2.1 Conceptual Design
개념 설계
  1. 1목적 — 다양한 형상 개념을 검토, 트레이드 스터디 수행, 최종 단일 최적 설계로 수렴
  2. 2설계 배치 항목 — 날개·꼬리 기하, 동체 형상, 승무원·페이로드·승객·장비의 내부 위치, 엔진 장착, 착륙 장치, 기타(canard? aft tail?)
  3. 3Fluid process — 설계 배치가 지속적으로 변경됨. 고급 CAD 시스템 불필요. 종이 스케치 수준으로 진행
  4. 4기간 — 수 주 ~ 6개월 (대형 상용·군용 프로젝트 기준)
  5. 5설계 상세도 — 전방 날개보(front wing spar)를 예로 들면, 개념 설계 수준에서는 루트~팁 간 평판(flat plate)으로만 표현. 날개 박스·연료 탱크·LE 플랩 크기 추정 가능
04 2.2.2 Preliminary Design
기본 설계
  1. 1시작 조건 — 주요 변경이 완료된 시점. Canard냐 후방 꼬리날개냐 등 큰 이슈가 해결된 후 시작
  2. 2형상 동결 (Freeze) — 전체 설계 배치 변경 금지. 이후 세부 변경만 허용 ★출제
  3. 3설계 성숙 — 수개월에 걸쳐 공력·중량·추진·안정성·구조 전문가들이 각 파트를 설계·분석. 신뢰도 증가
  4. 4Lofting — 핵심 작업. 외피를 충분한 정확도로 수학적으로 모델링. 서로 다른 설계자가 만든 부품들이 정확히 맞는지 확인 (wireframe → surface model) ★
  5. 5Mockup — 실물 크기 목업 또는 전자(CAD) 목업 제작. 실제 비용 추정 시작 ("You bet your company!")
  6. 6진지한 시험 시작 — 공력, 추진, 구조, 안정성·조종 분야 본격 시험
05 2.2.3 Detail Design
상세 설계
  1. 1실제 부품 설계 — 개별 구조 부품, 모든 시스템(착륙 장치·유압·전기·공압·연료·추진 등) 설계. 전투기: 수십만 부품; 대형 여객기: 수백만 부품 + 수백 마일의 배선
  2. 2생산 설계 — Production design: 제작 가능성·조립 순서 고려. 또 하나의 중요 업무
  3. 3시험 강화 — 실제 구조 시험, 비행 시뮬레이터 개발·운용 (제작사 + 고객 시험 조종사)
  4. 4시제기(Prototype) — "soft" 또는 임시 툴링으로 제작. 실제 생산용 hard tooling은 이후 단계
  5. 5실제 수치 확정 — "NOW you learn the real numbers!" 중량·성능 최종 확정
06 2.3 Conceptual Design
Process 3단계

① Conceptual Sketch
개념 스케치

  • 날개·꼬리 기하, 동체 형상, 주요 구성품 내부 위치(엔진·조종석·페이로드·착륙 장치·연료 탱크) 포함
  • 이전 설계 대비로 공력·중량 분율 추정 (Sizing)
  • 기존 설계와 유사하면 생략 가능

② Initial Sizing
초기 크기 결정

  • 설계 임무 수행에 필요한 총 중량과 연료 중량의 첫 번째 추정
  • 초기 배치도 작성 (단면 충분히 포함)
  • 배치도를 공력·중량·추진 분석에 사용 → 상세 크기 결정

③ Optimization through Iterative Process
반복 최적화

  • 가장 가볍거나 최저 비용이면서 임무 수행 + 성능 요구사항 충족하는 항공기 도출
  • 예: 조종면 전문가의 6자유도 해석으로 조종면 크기 검증 → 부족하면 설계자에게 확장 지시
  • 도면 번호 체계: D645-1 (Dash-One) → D645-2, D645-3 … D645-5 순으로 개정
  • "You never build the Dash-One" — 설계는 항상 반복된다
  • 최종 결과물은 기본 설계(Preliminary Design) 단계로 전달

기술 가용성: 근미래 → 현 기술만; 먼 미래 → 미래 기술 고려 가능(개발 위험 증가)

07 Fig. 2.4
개념 설계 흐름

개념 설계는 설계요구조건 + 새로운 아이디어에서 출발하여 두 단계를 거쳐 기본 설계로 넘어간다.

1차 루프 (초기 배치)

  • 개념 스케치 → 초기배치
  • 공력 / 중량 / 추진기관 초기 분석
  • 초기 크기 결정 및 성능 최적화
  • 불만족 시 → 개념 스케치로 되돌아감

2차 루프 (수정 배치)

  • 수정배치: 공력·중량·추진기관·비용·구조·착륙장치·기타 상세 분석
  • 수정된 크기 결정 및 성능 최적화
  • 불만족 시 → 1차 루프로 되돌아감
  • 만족 → 기본 설계(Preliminary) 진행
08 불확실성 하의 의사결정
7단계 기본 절차
  1. 1평가 기준 수립 — Establish evaluation criteria
  2. 2상대적 중요도 결정 — Determine relative importance of criteria
  3. 3대안 개념 개발 — 목표와 top-level requirements를 충족하는 대안 개념들 개발
  4. 4각 대안 평가 — 수립된 평가 기준에 따라 각 대안을 평가
  5. 5잠정 개념 선택 — 평가 기준을 가장 잘 만족하는 대안이 잠정적 개념 선택안이 됨
  6. 6잠정 선택안 상세 평가 — 예상치 못한 단점 식별을 위해 더 상세하게 평가
  7. 7최종 결정 또는 재검토 — 더 상세한 연구를 바탕으로 최종 결정. 필요시 3단계로 돌아감
09 AHP
분석적 계층 과정

Analytical Hierarchy Process (AHP)는 여러 평가 기준에 대한 쌍대 비교(pairwise comparison)를 통해 대안들의 우선순위를 정량적으로 결정하는 의사결정 기법이다.

Aij 비교값 의미:
Aij = 1 : i 와 j 가 동등하게 중요 (equal importance)
Aij = 3 : i 가 j 보다 약간 더 중요 (weakly more important)
Aij = 5 : i 가 j 보다 상당히 중요 (strongly more important)
Aij = 7 : i 가 j 보다 매우 중요 (very strongly more important)
Aij = 9 : i 가 j 보다 절대적으로 중요 (absolutely more important)
2, 4, 6, 8 → 중간 값

Aii = 1 (자기 자신과의 비교는 항상 1)
Aji = 1/Aij (역수 관계) ← if Aij = p, then Aji = 1/p ★ 역수 관계 암기

예시 (RFP 2011, 5대 속성): 연료 효율 > 신뢰성 > 속도 > 제자리비행 능력 > 비용

00 · 전기전자공학과의 역할

핵심 기여

항공우주(CFD)가 이론을 설계하고, 기계공학이 물리 구조를 만들면 — 전기전자는 실제 세계와 시스템을 연결합니다. 집안 곳곳의 센서를 네트워크로 묶고, 데이터를 AI 모델로 분석해 노즐과 팬을 실시간으로 구동합니다.

01 · 센서 선정 & 특성

센서 종류측정 대상추천 모듈인터페이스
PM2.5 / PM10미세먼지 농도Sensirion SPS30 (정밀) / GP2Y1014AU (저가)UART / I2C
CO₂ (NDIR)이산화탄소 농도SCD40 — ±40 ppm 정확도, 저전력I2C
VOC휘발성 유기화합물SGP30 또는 BME688 (복합 센서)I2C
온습도온도·상대습도SHT31 — ±0.2°C, ±2% RHI2C
풍속실내 기류 속도열선식 풍속계 또는 초음파식Analog / UART
PIR 동작 감지사람 존재 여부HC-SR501 — 재실 확인, 에너지 절약 모드Digital

센서 캘리브레이션

  • CO₂ 기준 캘리브레이션: 신선 외기(약 420 ppm)에서 영점 설정. SCD40은 자동 교정(ASC) 기능 활성화
  • PM2.5 보정: 환경 습도에 따라 광산란 값 오차 발생 → 습도 보정 계수 C_RH 적용
  • 온도 편차: ESP32 내부 발열로 온도 센서 +3~5°C 오차 발생 → 보드와 센서 이격 배치 또는 오프셋 보정

02 · IoT 시스템 아키텍처

센서 노드 (ESP32) ──MQTT──→ 브로커 (Pi) ──→ AI 예측 모델 ──→ 액추에이터

각 구역에 설치된 ESP32가 센서 값을 MQTT로 Raspberry Pi 브로커에 전송 → AI가 활동 패턴 분석 → 노즐 각도·팬 RPM 제어 명령 발행

MQTT 토픽 설계 예시

# 센서 데이터 발행 (ESP32 → 브로커)
aeroflow/sensor/living/pm25    → {"value": 23, "unit": "ug/m3"}
aeroflow/sensor/kitchen/co2    → {"value": 850, "unit": "ppm"}
aeroflow/sensor/bedroom/temp   → {"value": 22.4, "unit": "C"}

# 제어 명령 발행 (브로커 → 액추에이터)
aeroflow/control/nozzle/angle  → {"h": 30, "v": -15}  # 좌우·상하 각도
aeroflow/control/fan/rpm       → {"rpm": 1200}

ESP32 센서 읽기 코드 (예시)

// ESP32 + SCD40 CO₂ 센서 + MQTT 발행
#include <WiFi.h>
#include <PubSubClient.h>
#include <SensirionI2cScd4x.h>

void loop() {
  uint16_t co2; float temp, humi;
  scd4x.readMeasurement(co2, temp, humi);

  String payload = "{" +
    "\"co2\":" + co2 + "," +
    "\"temp\":" + temp + "}";

  client.publish("aeroflow/sensor/bedroom/co2", payload.c_str());
  delay(5000);  // 5초 주기 샘플링
}

03 · AI 예측 모델 — 선제 대응

핵심 아이디어

단순 반응(오염 감지 후 대응)이 아니라 예측(오염 발생 전 선제 대응)합니다. CO₂ 상승률의 시간 미분(dCO₂/dt)과 활동 시간대 패턴을 특징(feature)으로 사용해 "요리 시작 예측" 모델을 구현합니다.

특징 엔지니어링 (Feature Engineering)

특징(Feature)계산 방법의미
CO₂ 상승률dCO₂/dt = (C[t] - C[t-1]) / Δt급격한 상승 = 사람 활동 시작 신호
PM2.5 변화율dPM/dt (이동 평균 5 샘플)요리 시작 초기 연기 감지
시간대 원핫인코딩0~23시 → 24차원 벡터오전 7시·오후 6시 요리 패턴 학습
재실 여부PIR 센서 이진값 (0/1)재실 시만 집중 제어 활성화
VOC 급증SGP30 tvoc 임계값 초과 여부요리·청소 등 이벤트 감지

간단한 예측 모델 구현 (Python)

from sklearn.linear_model import LogisticRegression
import numpy as np

# 학습 데이터: [dCO2/dt, dPM/dt, 시간대, 재실]
X_train = np.array([...])
y_train = np.array([...])  # 0=평상시, 1=요리시작

model = LogisticRegression()
model.fit(X_train, y_train)

# 실시간 예측
def predict_cooking(co2_rate, pm_rate, hour, occupied):
    x = np.array([[co2_rate, pm_rate, hour, occupied]])
    prob = model.predict_proba(x)[0][1]
    if prob > 0.7:
        send_preemptive_control()  # 기류 선제 형성

04 · PWM 서보 & 팬 모터 제어

PWM 서보 신호
주기: 20 ms (50 Hz)
펄스 폭: 1.0 ms (0°) ~ 2.0 ms (180°)
Arduino: servo.write(angle) 또는 servo.writeMicroseconds(us)

팬 속도 제어 (BLDC + ESC)

  • ESC(Electronic Speed Controller): BLDC 팬 속도 제어. PWM 신호로 RPM 조절. 캘리브레이션 후 1000~2000µs 범위 사용
  • 피드백 제어: 풍속계 실측값을 아날로그 입력으로 읽어 PID 루프에 피드백
  • 저소음 PWM: 주파수 25 kHz 이상 사용 시 가청 노이즈 제거 (PWM 핀 타이머 설정)

S2
Wi-Fi + MQTT 통신 구현
Raspberry Pi에 Mosquitto 브로커 설치. ESP32에서 PubSubClient 라이브러리로 발행/구독 구현. Node-RED 대시보드로 실시간 시각화.
1~2주
S3
서보 + 팬 제어 코드 작성
Arduino Servo.h로 노즐 각도 제어. MQTT 수신 메시지 파싱 후 서보 각도 적용. 기계공학 팀 하드웨어와 통합 테스트.
1주
S4

통합 시스템 최종 테스트

2주

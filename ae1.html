    01 Nature of Aircraft Design
    IPD
    1. 1Multidisciplinary Systems Activity — 항공기 설계는 여러 분야가 참여하는 시스템 활동. 단계별 주요 업무와 분야 간 연관성 이해 필수
    2. 2Customer Requirements + Mission Goals — 고객 요구를 만족시키면서 임무를 성공적으로 수행하는 것이 설계의 목적
    3. 3IPD (Integrated Product Development) — 설계 초기 단계부터 운용 및 지원체계까지 동시적으로 고려하여 개발하는 통합개발 방법론 ★
    4. 4Systems Engineering — Aerodynamics / Structures / Propulsion / Controls + Mechanical, Electrical, Materials Engineering + Reliability, Safety, Human Factors, Logistics 등 통합
    5. 5Core Principle — Design ≠ Aero + Structures + Propulsion + … 단순 합이 아닌 체계적·종합적 사고 필요
    6. 6Requirements Conflict — "Light weight, high performance, large payload, easy to manufacture…" 각 요구사항은 서로 충돌. 한 분야 극대화 시 다른 분야 희생 (Dream Airplanes 개념)

    03 Development Phases
    Exploratory vs Full-Scale

    Exploratory Development

    • Conceptual Design + Preliminary Design 포함
    • Mission requirements, performance/cost goals 입력
    • Wind tunnel tests + CFD/structural/vibration simulations 병행

    Full-Scale Development

    • Detail Design + Prototype fabrication + Development test
    • CAD/CAM 활용
    • Flight test → Production
    04 Design Wheel
    Iterative Design Process

    1. 1Starting Point — 대부분 stated requirements에서 출발. 초기엔 군용: "Win the war", 민항: "Make money" 수준으로 모호함
    2. 2Requirements Refinement — 오랜 반복 과정을 거쳐 clean set of requirements 도출
    3. 3X-plane — 군/상업 목적 외에 개념 검증·실험 연구 목적으로 설계된 항공기 (concept validation aircraft)
    05 ConOps
    Concept of Operations
    1. 1Definition — 전체 시스템 수준의 운용 개념 및 절차를 정의한 문서 ★
    2. 2Scope — 항공기 설계뿐 아니라 Infrastructure, Traffic Management, Safety, Service까지 포함
    3. 3Role — 기체 설계 요구사항의 Top-Level Input Document (최상위 입력 문서)
    4. 4UAM Example — eVTOL + Vertiport + UTM/UATM + Ground Operations + Passenger Service + Communication/ATC + Maintenance + Regulation/Certification의 통합 시스템
    06 9-Step Conceptual Design
    Process
    #StepKey Content
    1Design RequirementsMission / Performance / Operational / Reliability / Maintainability requirements + Cost & Market analysis. Typical: Range, Payload, T-O/Landing distance, Maneuverability, Max speed, Stall speed, Structural limits
    2Concept SketchRequirements + New ideas → External configuration (Wing, Tail, Fuselage) + Internal layout (Seats, Engine, Cockpit) sketch. Used for aerodynamic analysis and weight estimation.
    3Comparative StudyCompare candidate configurations using Configuration Trade Study Matrix. Evaluation: Aerodynamics, Stability & Control, Weight, Propulsion, Structure, Subsystems, Systems Integration
    4Initial SizingUse historical data and statistical relationships. Select aircraft with minimum weight or cost satisfying all performance requirements.
    5Design / Analysis IterationFocus on external configuration design + conceptual subsystem layout → derive optimum baseline configuration
    6Baseline Configuration SelectionApply evaluation criteria + scoring matrix + weighting factors → determine optimum configuration
    7Subsystem Conceptual DesignConceptual design for each subsystem (Propulsion, Structure, Electrical, etc.)
    8Design Discipline ActivitiesSystems Integration, Configuration Design, Aerodynamic Analysis, Performance Analysis, Weight Analysis, Structural Design, Propulsion, Subsystems — each performed and results integrated
    9Cost EstimationProduction unit cost & life-cycle cost estimation. Goal is relative comparison, not absolute prediction.
    07 Preliminary Design Process

    ① Configuration Refinement

    • Iterative comparison of refined vs. baseline configurations
    • Wind tunnel tests + Computational analysis

    ② Structural Design

    • Design constraints (service life, g-limits) + External shape + Internal layout → Structural arrangement
    • Stress concentration and load analysis
    • Assembly drawings + Layout drawings (Landing gear, Engine)

    ③ Subsystem Design

    • Systems list compilation
    • Avionics, Electrical, Secondary Power, Hydraulic, Fuel, ECS, Landing Gear system layout drawings

    ④ System Development Plan

    • Budget/manpower plan, baseline configuration, development spec, design drawings
    • Configuration Management Plan, Test & Evaluation Plan
    08 Detail Design Process
    1. 1Baseline Configuration Elaboration — Sub-system → Component → Part level까지 세부화
    2. 2Specifications & Drawings — 관련 규격서, 성능 분석 결과, 세부 기술 도면 산출
    3. 3Software Development — Computer software definition and development
    4. 4Prototype Fabrication & Test — Prototype models fabricated; test and evaluation conducted
    09 Airworthiness & Certification

    Airworthiness ★

    • 항공기·부품이 특정 비행조건에서 정상 성능·안전성을 발휘하는 능력
    • Initial Airworthiness: 설계·제작 단계에서 안전기준 만족 확인
    • Continued Airworthiness: 운항 이후에도 안전성 유지 (정비·결함추적)

    Certification

    • Airworthiness를 규정과 절차에 따라 입증하고 당국이 승인/증서로 인정하는 법적 절차
    • 상업 항공기 개발·생산·운용 단계에서 적합성 확인

    Key Regulatory Bodies ★암기

    • FAA (USA): Federal Aviation Administration — FAR Part 23 (Small Aircraft), Part 25 (Transport Category)
    • EASA (Europe): European Union Aviation Safety Agency
    • ICAO (International): International Civil Aviation Organization
    • KCAS (Korea): Korean Civil Airworthiness Standards — based on FAA/EASA standards

이 프로젝트가 해결하려는 문제

왜 지금 이 가전인가?

우리는 하루 약 90%를 실내에서 보냅니다. 그런데 실내 공기는 외부보다 최대 5배 오염될 수 있습니다. 시중의 공기청정기는 '필터링' 방식에 머물러 있어, 기류가 닿지 않는 구석(데드존)은 영원히 오염된 채로 남습니다.

[DZ]
DEAD ZONE

소파 뒤, 구석, 커튼 뒤에 미세먼지가 쌓임. 필터 근처만 깨끗해지고 방 전체가 균일해지지 않음.

[PM]
요리 연기 급증

요리 시작 직후 PM2.5가 수백 µg/m³까지 치솟는데, 공기청정기는 이를 사후에야 감지. 선제 대응 불가.

[CO₂]
CO₂ 축적

밀폐 침실에서 CO₂가 1,500ppm 이상으로 상승해 수면 질 저하. 수면 중 수동 환기는 불가능.

[FX]
고정된 기류

에어컨·청정기는 좌우 스윙만 가능. 사람 위치와 가구 배치에 상관없이 동일한 패턴으로 바람을 보냄.

[NRG]
에너지 낭비

사람이 없는 공간에도 동일한 풍량 유지. 센서 기반 지능화 없어 불필요한 전력 소비.

[SOL]
해결책: 기류 설계

CFD + AI + 가변 노즐로 실내 유동장 자체를 지능 설계. 데드존 제거·선제 대응·균일 청정 달성.

2.2 OpenFOAM 단계별 실습 계획

왜 OpenFOAM인가?

무료 오픈소스 CFD 소프트웨어. 실제 연구·산업 현장에서 광범위하게 사용. HVAC·실내 환기 튜토리얼 풍부. Python으로 자동화 가능. ANSYS Fluent 대비 비용 없이 고급 기능 사용 가능.

핵심 차별점

드론 자세 제어(자세각 유지)와 Aero-Flow AX(실내 기류 방향 유지)는 동일한 수학적 프레임워크를 사용합니다. 상태 공간, 피드백 제어, 최적 제어이론이 그대로 적용됩니다.

3.1 상태 공간 표현 (State-Space)

항목Aero-Flow AX 정의
상태 변수 x각 구역의 [CO₂ 농도 C₁...Cₙ, PM2.5 농도 D₁...Dₙ, 유속 벡터 u₁...uₙ]
제어 입력 u각 노즐의 [상하 각도 θᵥ, 좌우 각도 θₕ, 팬 RPM n]
출력 yIoT 센서 실측값 (CO₂, PM2.5, 온도, 풍속)
시스템 방정식ẋ = Ax + Bu  (CFD 결과로 선형화한 A, B 행렬)
출력 방정식y = Cx + Du  (센서 배치에 따른 C 행렬)

3.2 PID 제어 (1단계 구현)

  • P (비례): 현재 CO₂ 농도와 목표값의 차이에 비례해 팬 RPM 증가 → 즉각 대응
  • I (적분): 장기간 오차 누적분 보정 → 정착 상태에서 잔류 오차 제거
  • D (미분): 농도 변화 속도 감지 → 요리 시작 같은 급격한 변화에 빠른 선제 반응
  • 구현 도구: Arduino PID 라이브러리로 서보모터와 팬 독립 제어

3.3 LQR 최적 제어 (핵심 차별점)

LQR 구성요소Aero-Flow AX 해석
상태 비용 행렬 QQ 값이 클수록 CO₂ 농도 편차를 엄격히 규제. 건강 기준치(1000ppm) 초과에 큰 패널티 부여.
제어 비용 행렬 RR 값이 클수록 노즐 작동 최소화 (에너지 절약). 소음·전력 소모 trade-off 조절.
리카티 방정식 (ARE)AᵀP + PA − PBR⁻¹BᵀP + Q = 0 을 풀어 최적 게인 K 도출
최적 제어 입력u* = −Kx  → 노즐 각도·팬 RPM을 최적으로 자동 결정
안정성 보장LQR은 설계상 닫힌 루프 시스템을 Lyapunov 안정성으로 보장

Python으로 LQR 구현 (scipy)

# LQR 최적 제어기 설계 — Aero-Flow AX
from scipy.linalg import solve_continuous_are
import numpy as np

# CFD 결과로 선형화한 시스템 행렬
A = np.array([...])  # 상태 천이 행렬
B = np.array([...])  # 제어 입력 행렬 (노즐·팬)

# 비용 행렬 설계
Q = np.diag([10, 10, 1, 1])  # CO₂·PM2.5 농도 편차에 큰 가중치
R = np.diag([1, 1])           # 제어 비용 (소음·전력 절약)

# 연속 Riccati 방정식 풀기
P = solve_continuous_are(A, B, Q, R)

# 최적 피드백 게인 계산
K = np.linalg.inv(R) @ B.T @ P
print('최적 게인 K:', K)

# 실시간 제어 루프
def control_step(sensor_readings):
    x = state_from_sensors(sensor_readings)
    u_star = -K @ x  # 최적 제어 입력
    return apply_to_nozzle(u_star)

3.4 MPC 예측 제어 — 선제 대응의 핵심

"요리 시작 전에 미리 기류를 형성"하는 원리

MPC(Model Predictive Control)는 N 스텝 앞을 내다보고 최적 제어 계획을 수립합니다. 드론의 경로 계획에서도 동일하게 사용되는 개념입니다. 평소 오후 6시에 요리 패턴이 있다면 → 5시 50분부터 주방 방향 기류를 사전 형성합니다.

  • 예측 지평선 (N): N 스텝 앞을 내다보고 최적 제어 계획 수립
  • 제약 조건 처리: 노즐 각도 한계 (±45°), 팬 최대 RPM, 소음 기준 등
  • 활동 패턴 학습 연동: CO₂ 상승률·온도 변화율의 시간 미분값을 특징(feature)으로 사용한 HMM/선형 회귀로 이벤트 예측

4.1 데드존 정량화 방법

지표정의 및 활용
공기 연령 (MAA)외부 공기가 특정 지점에 도달하는 평균 시간. 값이 클수록 데드존. CFD에서 스칼라로 직접 풀거나 후처리 계산.
환기 효율 지수 (ε)ε = τₙ/τₗ. 1보다 크면 완전 혼합 이상의 효율. 노즐 각도별 효율 비교에 사용.
국소 오염물 제거 효율오염원 위치별 배출 효율 비교. 어느 위치의 오염물이 가장 빠르게 배출되는지 지도 작성.
CFD 트레이서 가스법특정 지점에 가상 오염물 방출 후 농도 시간 변화 추적. 데드존 면적 지도(Heatmap) 생성 → 발표 핵심 그림.

4.2 노즐 유체역학 설계 포인트

  • 토출 속도 기준: 사람 방향 0.1~0.25 m/s (ASHRAE Standard 55 쾌적 기준), 데드존 방향 0.5~1.5 m/s
  • 연속 방정식 적용: 형상 변형 시 A₁v₁ = A₂v₂ 로 압력 손실 계산. 단면적 변화에 따른 출구 속도 보정
  • 코안다 효과 (Coanda Effect): 제트류가 근처 벽면을 따라 흐르는 현상. 천장 면을 이용해 기류를 멀리 보낼 수 있음
  • 자유 제트 이론: 중심선 속도 Uc ≈ 6.2Q/r. 거리에 따른 속도 감쇠를 노즐 방향 결정에 활용

핵심 수식 레퍼런스

A. 비압축성 나비에-스토크스 방정식

연속 방정식 (질량 보존)
∇·u = 0
운동량 방정식
ρ(∂u/∂t + u·∇u) = −∇p + μ∇²u + f

u = 속도 벡터  |  p = 압력  |  ρ = 밀도 (공기 ≈ 1.225 kg/m³)  |  μ = 동점성계수 (≈ 1.8×10⁻⁵ Pa·s)  |  f = 외력 (부력 등)

B. k-ε 난류 모델

난류 운동 에너지 k
∂k/∂t + u·∇k = ∇·[(ν + νₜ/σₖ)∇k] + Pₖ − ε
소산율 ε
∂ε/∂t + u·∇ε = ∇·[(ν + νₜ/σ_ε)∇ε] + C₁ε(ε/k)Pₖ − C₂ε(ε²/k)

표준 상수: C_μ = 0.09, C₁_ε = 1.44, C₂_ε = 1.92, σₖ = 1.0, σ_ε = 1.3

C. 레이놀즈 수

Reynolds Number
Re = ρuL/μ = uL/ν

예시: 노즐 출구 u = 0.5 m/s, L = 0.05 m → Re = (0.5 × 0.05) / (1.47×10⁻⁵) ≈ 1,700 → 층류/난류 전이 영역

D. LQR 최적화 목적함수

LQR 비용 함수
J = ∫₀^∞ [xᵀQx + uᵀRu] dt → 최소화
최적 피드백 게인
K = R⁻¹BᵀP      u*(t) = −Kx(t)

E. 공기 연령 (Mean Age of Air)

공기 연령 수송 방정식
∂(ρτ)/∂t + ∇·(ρuτ) = ∇·(ρD∇τ) + ρ

τ = 공기 연령 [s]. 값이 클수록 오래된 공기 = 데드존. OpenFOAM에서 passive scalar로 직접 풀거나 후처리 계산.

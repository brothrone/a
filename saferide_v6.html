02 / AI Model
YOLOv8 커스텀 클래스 (9개)

일반 YOLOv8(COCO 80종)을 직접 데이터 수집·라벨링하여 아래 9개 커스텀 클래스로 파인튜닝. TFLite 변환 후 Raspberry Pi 4B 온디바이스에서 각 카메라당 약 10fps 실시간 추론. 전면·후면 카메라를 루프 내에서 교번 순차 처리하여 100ms 1루프 안에 두 카메라의 모든 항목을 판별.

Class 3traffic_red적색 신호 (위반 감지)

03 / Operation
전체 작동 과정 (처음 → 끝)
PHASE 1

사용자 앱 — 잠금 해제 요청

▾

사용자가 공유 킥보드 앱에서 주변 킥보드를 선택하거나 QR 코드를 스캔하면 잠금 해제 요청이 발생합니다.

이 요청은 앱 → Wi-Fi(인터넷) → 클라우드 서버 → 킥보드 내 Raspberry Pi로 전달됩니다. Raspberry Pi가 신호를 수신하는 순간 카메라 시스템이 활성화되고 안전 검사 단계로 진입합니다.

PHASE 2

출발 전 안전 검사 — 후면 카메라

▾

잠금 해제 요청이 수신되면 후면 적외선 카메라가 탑승자를 촬영합니다. Raspberry Pi에서 YOLOv8이 추론을 실행하여 두 가지 조건을 동시에 검사합니다.

  • ⛑️
    헬멧 착용 감지
    YOLOv8이 helmet_on / helmet_off 중 하나로 분류. 3프레임 연속 helmet_off 감지 시 미착용으로 확정 (오탐 방지).
  • 👥
    탑승 인원 감지
    YOLOv8이 person_1 / person_2plus로 분류. 발판 위 사람 수를 바운딩 박스로 카운팅.
✅ 검사 통과

헬멧 착용 확인 + 1인 탑승 확인
→ ESC에 잠금 해제 신호 전송
→ 킥보드 출발 가능 상태

❌ 검사 실패

헬멧 미착용 또는 2인 이상 감지
→ ESC 잠금 유지
→ 벌점 부여 + 앱 경고 알림

PHASE 3

주행 중 실시간 감지 — 교번 순차 처리 + RPM

▾

Raspberry Pi 4B는 단일 보드이기 때문에 두 카메라를 진짜 동시에 처리하는 건 성능 부담이 큽니다. 대신 루프 내에서 전면·후면을 빠르게 번갈아(교번 순차) 처리하여 사실상 실시간처럼 동작합니다.

루프 1회 (약 100ms)
① 후면 카메라 프레임 캡처
② YOLOv8 추론 → 헬멧·인원 판별
③ 전면 카메라 프레임 캡처
④ YOLOv8 추론 → 노면·신호 판별
⑤ 위반 판단 → ESC 제어

1루프 = 약 100ms → 초당 10회 처리 → 각 카메라 기준 약 10fps
주행 중 노면·헬멧 상태는 수십 프레임 단위로 바뀌지 않으므로 10fps로 충분합니다.
  • 🛣️
    전면 카메라 — 노면 분류
    sidewalk(보도블록)가 5루프 이상 연속 감지되면 인도 주행으로 판정. 단일 루프 감지는 무시하여 오탐 방지. 인도 판정 시 ESC에 최대 5km/h 속도 제한 신호 전송.
  • 🚦
    전면 카메라 — 신호 인식 (개발 예정)
    traffic_red 감지 + RPM 속도 > 0 이면 신호위반으로 판정. 강제 정지 신호 전송.
  • 👥
    후면 카메라 — 주행 중 인원 모니터링
    출발 후 뒤에 사람이 올라타는 경우를 지속 감지. person_2plus 감지 시 즉시 경고음 + 속도 제한 + 벌점 처리.
  • ⚡
    RPM 센서 — 속도 측정
    BLDC 모터 RPM을 루프마다 읽어 현재 속도 계산. 제한 속도 초과 시 ESC에 감속 신호 즉시 전송. RPM은 카메라 추론과 독립적으로 매 루프 처리.

PHASE 4

위반 이벤트 생성 → Wi-Fi 전송 → 벌점 반영

▾

위반이 감지되면 Raspberry Pi가 즉시 위반 이벤트 패킷을 생성합니다. 패킷에는 위반 종류, 발생 시각, 사용자 ID, 감지 카메라 정보가 포함됩니다.

  • 📦
    이벤트 패킷 생성
    { type: "helmet_off", timestamp: "2026-04-27T20:15:00", user_id: "user_123", camera: "rear_IR" }
  • 📶
    Wi-Fi 전송 (프로토타입)
    Raspberry Pi 4B 내장 Wi-Fi (802.11ac)로 근거리 네트워크를 통해 클라우드 서버로 전송. 시험 환경에서는 Wi-Fi 커버리지 내에서 동작. 상용화 시 LTE로 교체 예정.
  • ☁️
    클라우드 서버 처리
    서버가 사용자 벌점 DB를 조회하고 해당 위반 벌점을 누적. 제재 기준 도달 시 제재 로직 자동 실행.
  • 📱
    앱 푸시 알림
    사용자 스마트폰에 위반 내용 + 현재 누적 벌점을 실시간 알림. 제재 단계 도달 시 이용 제한 안내.

PHASE 5

반납 및 세션 종료

▾

사용자가 앱에서 반납 처리를 완료하면 앱 → 서버 → Raspberry Pi 순으로 종료 신호가 전달됩니다.

  • 🔒
    ESC 잠금 전환
    Raspberry Pi가 ESC에 잠금 신호를 전송하여 킥보드를 잠금 상태로 전환. 카메라 및 추론 시스템 대기 모드로 전환.
  • 💾
    세션 데이터 저장
    해당 이용 세션의 위반 이력, 주행 시간, 감지 로그가 서버 DB에 저장. 다음 이용 시 누적 벌점이 그대로 유지.
벌점 초기화 조건:  6개월 무위반 시 전체 초기화  /  안전 교육 이수 완료 시 3점 차감

06 / AI Layer
AI 연계 단계
1단계 — 현재 구현

실시간 감지 + 즉시 제재

YOLOv8 커스텀 파인튜닝으로 헬멧·인원·노면·신호를 단일 패스 탐지. 위반 즉시 잠금·속도 제어 + 벌점 부여.

2단계 — 확장 방향

위반 패턴 학습 → 예측 제어

사용자별 위반 이력 누적 학습(LSTM). 고위험 사용자 선제 경고, 구역별 히트맵 생성, 보험사 연계 할인 모델.
